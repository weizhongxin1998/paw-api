# AGENTS.md — Paw API

## Principles

Every design and code decision must favor **maintainability, extensibility, low memory usage, and high performance** — in that order. Avoid premature abstraction, but keep layers clean for future features. SQLite queries, HTTP clients, and reactive frontend state must not leak or hold references longer than needed.

## Project

Wails v2 desktop app (Go 1.23 + Vue 3 + TypeScript + Vite). A local API debugging/management tool (no accounts, no cloud). Still at template scaffold stage.

## Commands

| Action | Command | Notes |
|---|---|---|
| Live dev | `wails dev` | Starts Go backend + Vite HMR frontend |
| Production build | `wails build` | Outputs to `build/bin/` |
| Frontend dev only | `npm run dev` | Run from `frontend/` |
| Frontend build | `npm run build` | Runs `vue-tsc --noEmit && vite build` |
| Frontend preview | `npm run preview` | Vite preview of built frontend |

Frontend must be installed first: `npm install` in `frontend/`.

## Architecture (planned — see `docs/spec/paw-api-design.md`)

```
Handlers (Wails Bind) → Services → Repositories → SQLite (modernc.org/sqlite, no CGO)
```

Desired Go packages: `models/`, `database/`, `repositories/`, `services/`, `handlers/`, `pkg/httpclient/`.

Frontend: Vue 3 + Pinia stores + Composables, under `frontend/src/views/`, `components/`, `stores/`.

## Generated code — do not edit

- `frontend/wailsjs/go/main/App.js` and `App.d.ts` — auto-regenerated by `wails dev` or `wails generate module`.
- `frontend/wailsjs/runtime/` — Wails runtime JS.

## Go specifics

- `go.mod` has a local `replace` directive for Wails — do not remove.
- `main.go` embeds `frontend/dist` via `//go:embed all:frontend/dist`.
- `App` struct methods in `app.go` are bound as Wails-callable APIs.

## Frontend specifics

- Vue 3 `<script setup>` SFCs.
- Frontend build includes type-checking (`vue-tsc --noEmit`) before bundling.
- No linting, formatting, or test tooling configured yet.

## Tests

None exist yet. No test framework, no CI. Frontend has no test scripts.

## Git

`.gitignore` covers: `build/bin`, `node_modules`, `frontend/dist/`.
